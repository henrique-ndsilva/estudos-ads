1. São funcionalidades características do kernel de um sistema operacional:

  1. interpretar as solicitações dos usuários, interpretar comandos.
  2. assumir o controle do programa, atuar sobre as execuções dos usuários.
  3. interpretar os comandos dos usuários, gerenciar a memória.
  4. processar as interrupções, assumir o controle do programa.
  5. escalonar as tarefas, cuidar da comunicação entre os processos.

1.1 Os sistemas operacionais podem ser monoprogramáveis/monotarefa ou multiprogramáveis/multitarefa. Assinale a alternativa correta:

  1. Independentemente de os sistemas operacionais serem monoprogramáveis ou multiprogramáveis, ambos executam várias tarefas, seguindo um escalonamento da tarefa mais prioritária para a de menor prioridade.
  2. Os sistemas multiprogramáveis foram implementados nos mainframes e não são utilizados em computadores atuais.
  3. Os sistemas multiprogramáveis utilizam os recursos do processador para uma tarefa específica e, com isto, apresentam um rendimento maior.
  4. Os sistemas monoprogramáveis são caracterizados pelo uso de processador, memória e dispositivos periféricos, de forma dedicada a um único usuário.
  5. Os sistemas monoprogramáveis são sistemas mais atuais, pois dedicam mais tempo para a execução de processos prioritários e, com isto, oferecem maior velocidade para as aplicações.

2. Um servidor de aplicação, instalado em uma infraestrutura de TI de uma empresa, tem por função:

  1. gerenciar as aplicações de acesso à base de dados e, com isto, manter os arquivos dos bancos de dados mais seguros e, assim, evitando riscos de roubo de dados, por exemplo.
  2. facilitar a instalação de novas aplicações, pois ele serve para manter todos os arquivos executáveis e assim funcionar como um repositório.
  3. gerenciar os serviços da aplicação de impressoras e, com isto, diminuir os custos com várias impressoras espalhadas na empresa.
  4. organizar todos os arquivos das aplicações dos usuários separados por pastas e assim o acesso ficará mais controlado e seguro.
  5. facilitar o desenvolvimento do software e para isto proporciona uma interface entre o software da aplicação de negócio e as características complexas que são comuns para a maioria das aplicações.

3. Os provedores de Plataforma como Serviços (PaaS) têm diversos serviços e características. Entre eles podemos citar:

  1. Os provedores de PaaS possuem diversos serviços que proporcionam o desenvolvimento de aplicativos. O pagamento deve ser feito pelo licenciamento de número de licenças de software e que incluem a atualização anual.
  2. Desenvolvimento de aplicações Web com diversas linguagens e com o provisionamento de sistema operacional, servidores e balanceamento de carga.
  3. Os provedores de PaaS fornecem as máquinas virtuais, os sistemas operacionais e os bancos de dados, e os clientes devem instalar as ferramentas de desenvolvimento.
  4. Implantação de sistemas de ERP e, assim, os usuários ficam responsáveis por lançamento dos dados de suas aplicações.
  5. Os provedores de PaaS ficam responsáveis pelo desenvolvimento de aplicações de nível empresarial.

3.1 A Plataforma como serviço – PaaS é uma solução fornecida pelos provedores de serviços de nuvem. Quando comparada com a TI tradicional muitas das responsabilidades passam a ser do fornecedor de nuvem. Selecione das alternativas abaixo quais as responsabilidades que ficam para o cliente de PaaS:

  1. O cliente de PaaS deve se responsabilizar pela instalação dos ambientes de middleware para os contêineres de seus projetos.
  2. O cliente de PaaS deve se responsabilizar por instalar suas ferramentas para o desenvolvimento de aplicações e, também, para cuidar da aquisição das licenças sob demanda.
  3. O cliente de PaaS só utiliza os aplicativos que são fornecidos pelo provedor e sua responsabilidade será só a entrada dos dados, como, por exemplo, nos sistemas de Gestão de Negócios.
  4. O cliente de PaaS deve focar no desenvolvimento de suas aplicações com a utilização das ferramentas de programação, dos bancos de dados e outros serviços de desenvolvimento fornecidos pelo provedor.
  5. O cliente de PaaS deve se responsabilizar pela aquisição dos sistemas operacionais para que possa instalar nas máquinas virtuais fornecidas pelo provedor.

4. Ao transmitir os dados de um computador para outro através de uma rede de comunicação, tornou-se necessária a divisão das tarefas em etapas, as quais foram chamadas de camadas. Selecionar a alternativa correta quanto a atividade realizada na camada.

  1. A camada de rede junto com a camada de transporte possui a responsabilidade de verificar a segurança dos dados dos usuários e só encaminhar caso tenham sido validados.
  2. A camada de enlace é responsável por endereçar os dispositivos envolvidos na conexão, e faz isso com a utilização de um endereço lógico, que pode variar de hardware para hardware.
  3. A camada de rede é responsável por receber os dados da aplicação do usuário e colocá-los para a camada de transporte que se encarregará de enviar para o outro dispositivo de destino.
  4. A camada de aplicação é responsável por receber os dados dos usuários, verificar o endereço físico do destino desejado, e só então disponibilizar para a camada de transporte.
  5. A camada de transporte é responsável pelas funções de conexão entre os dispositivos fonte e destino, segmentando os dados em unidades menores.

4.1 As redes VLANs podem ser utilizadas para aumentar a performance da rede separando os grandes domínios de broadcast em domínios menores e oferece características importantes. Selecione a resposta correta:

  1. Com relação à segurança, os grupos de usuários com informações confidenciais são colocados em redes separadas.
  2. As redes VLANs diminuem o domínio de broadcast, porém aumenta o tráfego de atualização dos dados dos computadores.
  3. Uma das preocupações fundamentais para o sucesso das redes VLANs é dar nomes adequados, e isso diminuirá o tráfego.
  4. A divisão da rede em grupos de trabalho, apesar de ser mais fácil de projetar, acarreta tráfego muito maior.
  5. As redes VLANs são importantes para a diminuição do tráfego, porém, o gerenciamento dos aplicativos é mais difícil.

5. Uma empresa utiliza em seus escritórios o processamento de texto através do Google Docs, e o departamento de mídia trabalha com um ambiente de edição de vídeos hospedado na nuvem. Com essas informações, selecione a alternativa correta:

  1. Na computação em nuvem, esse é um serviço do tipo PaaS.
  2. Na computação em nuvem, esse é um serviço do tipo IaaS.
  3. Na computação em nuvem, esse é um serviço do tipo SaaS.
  4. Na computação em nuvem, o primeiro serviço pode ser SaaS e o segundo não se encaixa em ambiente de nuvem.
  5. Na computação em nuvem, esse é um serviço que não se encaixa em ambientes de nuvem, pois edição de vídeos demanda muito a utilização de rede.

5.1 O servidor DHCP é utilizado para facilitar a atribuição de endereços IP para as estações. Assinale a alternativa correta quanto às características do DHCP:

  1. Os endereços IP atribuídos para as estações de uma rede são feitos de forma aleatória.
  2. O servidor DHCP aloca endereços IP para os hosts e para os servidores, assim, os administradores não se preocupam com endereçamentos.
  3. A alocação dos endereços IP através do DHCP, apesar de serem interessantes para o administrador de redes, causa muito tráfego na rede.
  4. O servidor DHCP possui mecanismos de alocação de endereços, que podem ser manuais, automáticos e dinâmicos.
  5. O servidor DHCP é importante para que os endereços MAC possam ser alocados dinamicamente.

6. Com relação à virtualização de servidores, vários conceitos são utilizados. Assinale a alternativa correta:

  1. A virtualização de servidores tem por objetivo aumentar o tamanho físico dos servidores e, assim, poder disponibilizar mais hardware.
  2. Com a virtualização de servidores, os tempos ociosos da CPU são completamente eliminados e, assim, ocorre o aproveitamento de 100% da capacidade.
  3. A virtualização de servidores só é possível para computadores que tiverem o componente de hardware chamado hypervisor.
  4. A capacidade de servidores serem transferidos para outros servidores físicos é conhecida como instância.
  5. No processo de virtualização, os computadores físicos são convertidos em máquinas virtuais e isso apenas com relação ao processador.

6.1 Nos sistemas dos mainframes, os discos rígidos são utilizados com a tecnologia RAID – redundant array of independent disks. Com relação à utilização dessa tecnologia, podemos afirmar que ela:

  1. aumenta a performance dos cálculos matemáticos complexos, pois uma parte dos dados da equação fica em um disco, e o resultado parcial, em outro. Assim o acesso fica mais rápido.
  2. é implementada em um disco rígido e, por ser uma velocidade de acesso, é mais rápida.
  3. gerencia a velocidade dos discos e procura sempre gravar no disco mais rápido.
  4. gerencia o armazenamento, de forma a garantir que os dados não sejam comprometidos, mesmo quando acontecerem falhas.
  5. garante que os mainframes possam ser utilizados em qualquer tipo de atividade, desde uma pequena empresa até uma empresa com muitas filiais, por exemplo.

6.2 Um Data Center tem um papel importante nos atuais cenários de infraestrutura de TI e para os negócios das empresas e, para isto, torna-se necessário um dimensionamento dos serviços a serem oferecidos e que devem seguir determinados critérios. Selecione a resposta correta, quanto aos critérios a serem analisados quando da escolha de um Data Center:

  1. Os critérios a serem avaliados devem fornecer indicadores de desempenho, disponibilidade, escalabilidade, segurança e gerenciabilidade.
  2. Os critérios a serem avaliados envolvem a condição de os equipamentos continuarem funcionando, independentemente de algum incidente, além das condições dos softwares de controle sem riscos de invasão ou mesmo de vírus.
  3. Os critérios a serem avaliados devem fornecer indicadores de qualidade das instalações elétricas, qualidade dos fornecedores dos servidores e equipamentos de rede e condição financeira da empresa proprietária do Data Center.
  4. A empresa proprietária do Data Center deve ser avaliada quanto à sua capacidade técnica e financeira e estar instalada em uma região livre de terremotos ou inundações.
  5. A empresa proprietária do Data Center deve ser de alta confiança, a começar pelos sócios e o local de instalação física, evitando crises ocasionadas por instabilidade do tempo.

7. Uma empresa em crescimento possui uma rede com 20 computadores e um switch de 24 portas que os interliga. Uma área de negócios com mais 15 usuários foi criada e ficará no mesmo andar, formando então a rede nova. Especifique os equipamentos de rede que você utilizará, sendo que um dos requisitos será que todos os computadores da empresa possam se comunicar. Justifique as escolhas dos equipamentos.

Resposta: Utilizaria mais um switch para conectar os 15 novos computadores e um roteador para interligar as redes, garantindo que todos os computadores possam se comunicar. Justifica-se o uso do switch para conexões locais e do roteador para comunicação entre redes.

Explicação: A justificativa da escolha do switch e do roteador é que a função básica de cada um deles é bem definida, sendo o switch usado para interligar computadores na camada de enlace, e o roteador, para interligar redes.

7.1 Faça uma comparação do modelo de SaaS com a arquitetura tradicional de TI e explique como os desenvolvedores de aplicações podem usar o conceito de Marketplace para comercializar seus produtos e serviços.

Resposta: Diferente da arquitetura tradicional, onde o software é instalado localmente, o modelo SaaS oferece acesso via internet, sem necessidade de infraestrutura própria. Desenvolvedores podem usar Marketplaces para divulgar, vender e distribuir seus aplicativos de forma mais rápida e escalável.

Explicação: O aluno deve demonstrar conhecer a infraestrutura de serviços de TI e o posicionamento de SaaS nessa arquitetura. E quanto à utilização das empresas fornecedoras de software, será interessante mostrar o conhecimento de Marketplace.
