1. Quais são os dois principais itens que são verificados na análise de viabilidade de um projeto de sistema?

Resposta Correta: Financeiros e técnicos.

2. Um diretor da área de desenvolvimento, com perfil tradicional e preocupado com os objetivos finais do software, sugeriu um modelo que combinasse os fundamentos da prototipação com os aspectos sistemáticos e controlados do modelo "cascata".

Qual o modelo sugerido pelo diretor de desenvolvimento?

Resposta Correta: Espiral

3. No algoritmo descrito abaixo, representado em pseudocódigo, que tipo de estrutura de decisão está sendo utilizada? Assinale a alternativa correta.

Resposta Correta: Estrutura de decisão composta.

4. O plano de gerenciamento de riscos é um dos planejamentos mais importantes que o gerente de projeto deve elaborar.

Qual das alternativas abaixo descreve os riscos mais importantes que devem ser mapeados?

Resposta Correta: Projeto, produto e negócio.

5. Um sistema da Empresa Alfa foi acessado indevidamente por usuários maliciosos e causaram prejuízos econômicos para a empresa.

Qual das alternativas abaixo descreve o principal atributo do sistema que se mostrou frágil?

Resposta Correta: Confiança e proteção.

6. O diretor de tecnologia solicitou ao gerente do projeto de desenvolvimento do software de monitoramento de remédios controlados que o algoritmo fosse representado graficamente.

Indique a alternativa que corresponde às representações gráficas de algoritmos:

Resposta Correta: Fluxograma e diagrama de Chapin.

7. O presidente de uma rede de drogarias recomendou ao diretor de tecnologia, com relação ao desenvolvimento do software de monitoramento de remédios controlados, muita atenção no gerenciamento de riscos que pudessem afetar a organização.

Qual alternativa indica o tipo de risco a que o presidente se refere?

Resposta Correta: Risco de negócios.

8. O presidente de uma rede de drogarias solicitou o desenvolvimento de um software para monitorar a venda de remédios controlados. O diretor da área de tecnologia da empresa solicitou ao gerente de projetos atenção nas tarefas de atribuição de prazos, responsáveis e recursos para cada atividade do projeto.

Qual alternativa representa a atividade de gerenciamento de projetos a que o diretor de tecnologia se refere?

Resposta Correta: Programação de atividades.

9. Durante a fase de levantamento de requisitos, o diretor de tecnologia sugeriu que fossem realizadas reuniões com os principais envolvidos no projeto, principalmente para evitar requisitos contraditórios e promover acordos entre os solicitantes.

Qual é a alternativa que indica a técnica sugerida pelo diretor?

Resposta Correta: Técnica de levantamento em grupo.

10. O gerente do projeto solicitou que no diagrama de casos de uso fosse identificada a sequência na qual os casos de uso precisam ser desenvolvidos.

Que tipo de relacionamento deve ser utilizado para atender à solicitação do gerente de projetos?

Resposta Correta: Dependência

11. Em uma reunião de controle do andamento do Projeto Pegasus, referente ao desenvolvimento de um novo software, o gerente de Projetos estava mais preocupado com as respostas para as questões do tipo?

Resposta Correta: Quanto tempo estamos atrasados? Qual o impacto no orçamento?

12. Na representação de um algoritmo, por meio do pseudocódigo, qual o comando principal caracteriza uma estrutura de decisão composta?

Assinale a alternativa correta.

Resposta Correta: "senão "

13. Entre as técnicas principais utilizadas para o levantamento de requisitos, o diretor executivo sugeriu adotar a utilização de questionários, realização de entrevistas e análise de documentação.

A qual o tipo de técnica de identificação de requisitos o diretor se referiu?

Resposta Correta: Técnicas tradicionais

14. Para o desenvolvimento do Projeto Pegasus, de desenvolvimento de software, a Diretoria de Tecnologia da Informação (TI) determinou que fosse adotada uma estratégia na qual os componentes do novo software fossem desenvolvidos como unidades independentes.

Qual paradigma está mais associado a essa estratégia?

Resposta Correta: Orientado a Objeto.

15. O produto gerado pelo Projeto Pegasus está em produção, iniciando a nova etapa do ciclo de vida do desenvolvimento: a manutenção. Porém, após a fase de testes e antes da implantação, o sistema operacional de seu servidor foi alterado sem a devida comunicação ao gestor do projeto, trazendo problemas de lentidão ao produto.

Qual o motivo principal e que tipo de manutenção deve ser aplicada?

Resposta Correta: Alterações no ambiente de processamento: manutenção adaptativa.

16. Qual alternativa descreve o modelo de desenvolvimento em que o software é produzido em uma série de versões evolucionárias e reconhece abertamente as possibilidades de algo dar errado?

Resposta Correta: Espiral.

17. A equipe do projeto do desenvolvimento do software solicitou que fosse desenhado o diagrama de classes.

Qual destas alternativa representa o principal objetivo do diagrama de classes?

Resposta Correta: Fornecer uma representação gráfica da estrutura estática do software.

18. Durante o desenvolvimento do Projeto Pegasus, o gerente de projeto demonstrou bastante atenção com o cuidado e o gerenciamento de todos os recursos humanos envolvidos. Qual a alternativa que descreve a área de preocupação do gerente do projeto?

Resposta Correta: Pessoal.

19. Na lógica de programação, é constante a necessidade da tomada de decisão representada por meio de estruturas de decisão. Assinale a alternativa que corresponde à estrutura de decisão que executa comandos a partir apenas da condição verdadeira e que apresenta três operadores relacionais.

Resposta Correta: Estrutura de decisão simples / "=", "<" e ">".

20. Um diretor da área de desenvolvimento determinou que o processo para a identificação dos requisitos fosse cumprido corretamente. Qual a correta sequência de atividades do processo de engenharia de requisitos?

Resposta Correta: Concepção > Levantamento > Elaboração > Negociação > Especificação > Validação

21. Um diretor da área de desenvolvimento, preocupado com a quantidade de funções solicitadas para o novo software, recomendou à sua equipe de testes que fossem realizados alguns testes, de forma integrada, sempre que um novo componente fosse adicionado à solução. A qual tipo de teste o diretor está se referindo?

Resposta Correta: Regressão

22. O diretor executivo de uma empresa de seguros solicitou o desenvolvimento de um software para identificar, no mercado, os veículos com seguros de outra empresa, com a finalidade de ofertar o seu produto. O diretor da área de desenvolvimento recomendou ao gerente de projeto uma atenção especial à atividade de levantar o esforço humano exigido, a duração cronológica do projeto e os custos. A qual atividade de gerenciamento de projeto o diretor de TI está se referindo?

Resposta Correta: Estimativa

23. Um novo projeto de desenvolvimento de software foi iniciado na Empresa Alfa, em que o cliente conhecia bem seus objetivos mas não tinha claramente definido algumas funcionalidades e recursos. Qual das alternativas descreve o melhor modelo a ser utilizado no projeto?

Resposta Correta: Prototipação.

24. A metodologia denominada "DevOps" foi concebida pela junção dos processos das áreas de desenvolvimento e operações em uma única metodologia. Qual a alternativa descreve melhor suas características?

Resposta Correta: Colaboração entre áreas, implantação contínua e compartilhamento de recursos.

25. Processo bem disciplinado, abordagem preditiva, a elaboração de documentação abrangente e o uso de ferramentas de apoio são características de uma metodologia de desenvolvimento de projeto. Indique a alternativa que descreva essa metodologia.

Resposta Correta: Tradicional.

26. Na representação de um pequeno trecho do diagrama de caso de uso de um software de monitoramento de remédios controlados, os "bonecos-palitos" que retratam o atendente da drogaria, o gerente da drogaria e o cliente, representam qual elemento de um diagrama de caso de uso? Assinale a alternativa correta.

Resposta Correta: Atores.

27. Durante a etapa de construção, é realizada a concepção da modularidade da estrutura do programa. Qual das alternativas abaixo representa os 3 principais tipos de módulos?

Resposta Correta: Sequencial, incremental e paralelo.

28. Assinale a alternativa que contém três elementos de um diagrama de atividades:

Resposta Correta: Ação, fluxo e decisão.

29. Qual dos itens abaixo apresenta três características de software?

Resposta Correta: Complexidade, mutabilidade e conformidade.

30. Como o software solicitado pelo diretor executivo envolve o levantamento de informações de clientes de outra seguradora, o diretor determinou que os requisitos legais e éticos sejam bem detalhados para garantir que o novo software fique em conformidade com a legislação e seja aceito pelo público em geral. Com qual o tipo de requisito não funcional o diretor executivo está preocupado?

Resposta Correta: Requisitos externos

31. Um sistema de uma empresa de seguros foi desenvolvido e entregue. Porém, depois de dois meses em produção, ficou quase sem nenhuma utilização, apenas 10% das suas funções continuaram necessárias. Qual etapa do ciclo de desenvolvimento de software não foi bem realizada?

Resposta Correta: Análise

32. Considerado o modelo de ciclo de vida clássico ou tradicional de desenvolvimento de software, quais são e como estão, genericamente, sequenciadas suas etapas?

Resposta Correta: Análise > Projeto > Construção > Testes > Implantação > Manutenção.

33. O gerente do projeto da rede de drogarias é responsável pelo desenvolvimento de um outro software, o qual está mais adiantado, realizando as atividades de especificação da estrutura de dados, procedimentos de operação, arquitetura de software e definição de interfaces. Qual alternativa corresponde à etapa do ciclo de desenvolvimento em que o software se encontra?

Resposta Correta: Projeto.

34. Qual alternativa representa as diferenças entre as metodologias ágeis e tradicionais, considerando os atributos de cultura, orientação e medição de sucesso?

Resposta Correta: Ágil: liderança, pessoas, valor para o negócio. Tradicional: comando, processo, conformidade com o planejamento.

33. No diagrama de casos de uso, elaborado para o software solicitado, foram desenhados os vendedores que irão utilizar o software para identificar os clientes que ofertarão o novo seguro. Os vendedores são elementos do diagrama conhecido como

Resposta Correta: Atores.

36. Os modelos iterativos de desenvolvimento de software são considerados modelos evolucionários. Qual das alternativas abaixo representa os principais modelos iterativos corretamente?

Resposta Correta: Prototipação e espiral.

37. O diretor da área de tecnologia demonstrou uma preocupação com a programação dos algoritmos para o software, já que, no passado, alguns softwares apresentaram falhas de programação. Qual alternativa indica que tipo de teste deve ser realizado com mais atenção?

Resposta Correta: Unidade.

38. Durante a etapa dos testes do Projeto Pegasus, adicionou-se um novo componente ao sistema e identificaram-se várias falhas. Qual tipo de teste foi realizado para identificação desse problema?

Resposta Correta: Regressão.
